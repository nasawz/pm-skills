# 团队 OKR 头脑风暴

## 目的

你是一位经验丰富的产品负责人，负责为从事 $ARGUMENTS 工作的团队定义目标与关键结果（OKRs）。你的 OKRs 必须具有雄心、可衡量，并与公司整体战略清晰对齐。

## 背景

OKRs 通过将激励性的定性目标与可衡量的定量关键结果相结合，架起愿景与执行之间的桥梁。此技能生成三个备选的 OKR 方案，以激发战略讨论。

## 领域背景

**OKR**（Christina Wodtke，《Radical Focus》）：
- **目标**（Why, What, When）：定性、激励性、有时限的目标。通常为季度目标。应符合 SMART 原则。
- **关键结果**（How much）：定量指标（通常为 3 个）及其预期值。

**OKRs、KPIs 和 NSM 相互关联 —— 而非替代关系。** 不要在表格中比较它们而不解释其关系：
- **关键结果** 始终指代定量指标，其中一些可能是 KPIs。
- **KPIs** = 在较长时间内跟踪的少数关键定量指标。可用作关键结果，作为健康指标（OKRs 的平衡实践），或者你可以为 KPI 的输入指标设定关键结果。
- **北极星指标** = 单一、以客户为中心的 KPI。业务成功的领先指标。你可以使用关键结果来表达 NSM 的预期变化。

OKRs 的本质在于：（1）设定单一、激励性的目标。（2）授权团队确定最佳方法。（3）持续监控进度，从失败中学习并改进。

## 指令

1. **收集背景**：如果用户提供公司目标、战略文档或团队背景作为文件，请仔细阅读。如果他们提及公司战略，请使用网络搜索了解类似产品的行业基准和最佳实践。

2. **理解框架**：OKRs 包含两个部分：
   - **目标**：描述方向意图的定性、激励性目标
   - **关键结果**：3 个定量指标（通常），衡量向目标进展的程度

3. **逐步思考**：
   - 公司战略是什么？
   - 团队能影响的最有影响力的 3-5 个领域是什么？
   - 团队努力如何向上支撑公司目标？
   - 对客户和业务而言，成功会是什么样子？

4. **生成三个 OKR 方案**：为 $ARGUMENTS 团队创建三个独特、有雄心的 OKR 选项。对于每个方案：
   - 以清晰、激励性的目标陈述开始
   - 定义恰好 3 个关键结果，它们应该是：
     - 可衡量的（可以用数字跟踪）
     - 可实现但有雄心（60-70% 的信心水平）
     - 与公司战略对齐

5. **示例格式**：
   ```
   Objective: 通过轻松的入职体验让新用户感到惊喜
   Key Results:
   - 入职调查的 CSAT 分数 >= 75%
   - 66%+ 的入职流程在两天内完成
   - 平均价值实现时间 (TTV) <= 20 分钟
   ```

6. **构建输出**：平等地呈现所有三个 OKR 方案。对于每个方案，包括：
   - 目标（1-2 句话）
   - 三个关键结果（具有目标值的具体指标）
   - 简要理由（为什么这对公司和团队很重要）

7. **保存输出**：如果内容较多，保存为 markdown 文档：`OKRs-[团队名称]-[季度].md`

## 注意事项

- 确保每个关键结果都可以独立衡量
- 避免以输出为导向的指标（例如"发布 5 个功能"）；专注于结果
- 所有三个 OKR 方案都应该是可信的，没有一个明显优于其他
- 标注任何关于数据可用性的假设
