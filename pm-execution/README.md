# pm-execution

执行与产品管理技能：PRD（产品需求文档）、OKR（目标与关键结果）、路线图、冲刺、事前分析、利益相关者地图、用户故事、优先级框架等。

## 技能（15项）

- **brainstorm-okrs** — 头脑风暴团队级 OKR，与公司目标保持一致。
- **create-prd** — 使用全面的 8 部分模板创建产品需求文档，涵盖摘要、背景、目标、市场细分、价值主张、解决方案详情和发布计划。
- **dummy-dataset** — 生成用于测试的真实测试数据集，支持可自定义的列、约束条件和输出格式（CSV、JSON、SQL、Python 脚本）。
- **job-stories** — 使用"当[情境]时，我想要[动机]，以便我能够[结果]"的格式创建工作故事，包含详细的验收标准。
- **outcome-roadmap** — 将以产出为导向的路线图转换为以成果为导向的路线图。
- **pre-mortem** — 对 PRD 进行事前分析。
- **prioritization-frameworks** — 9 种优先级框架参考指南，包含公式、使用场景指导和模板。
- **release-notes** — 从工单、PRD 或变更日志生成面向用户的发布说明。
- **retro** — 主持结构化的冲刺回顾会议。
- **sprint-plan** — 规划冲刺，包括产能估算、故事选择、依赖关系映射和风险识别。
- **stakeholder-map** — 使用权力/利益矩阵构建利益相关者地图，识别每个象限的沟通策略，并生成沟通计划。
- **summarize-meeting** — 将会议纪要总结为结构化模板，包含日期、参与者、主题、要点和行动项。
- **test-scenarios** — 从用户故事创建全面的测试场景，包含测试目标、起始条件、用户角色、分步操作和预期结果。
- **user-stories** — 遵循 3C 原则（卡片、对话、确认）和 INVEST 标准创建用户故事，包含描述、设计链接和验收标准。
- **wwas** — 以 Why-What-Acceptance（为什么-做什么-验收标准）格式创建产品待办事项。

## 命令（10个）

- `/pm-execution:generate-data` — 生成用于测试的真实测试数据集 — CSV、JSON、SQL 插入语句或 Python 脚本。
- `/pm-execution:meeting-notes` — 将会议纪要总结为结构化笔记，包含决策、行动项和后续跟进。
- `/pm-execution:plan-okrs` — 头脑风暴团队级 OKR，与公司目标保持一致 — 定性目标和可衡量的关键结果。
- `/pm-execution:pre-mortem` — 对 PRD、发布计划或功能进行事前风险分析 — 在问题发生前识别可能出错的地方。
- `/pm-execution:sprint` — 冲刺生命周期 — 规划冲刺、运行回顾会议或生成发布说明。
- `/pm-execution:stakeholder-map` — 在权力 × 利益矩阵上映射利益相关者，并创建定制的沟通计划。
- `/pm-execution:test-scenarios` — 从用户故事或功能规格生成全面的测试场景 — 正常路径、边界情况和错误处理。
- `/pm-execution:transform-roadmap` — 将基于功能的路线图转换为传达战略意图的以成果为导向的路线图。
- `/pm-execution:write-prd` — 从功能创意或问题陈述创建全面的产品需求文档。
- `/pm-execution:write-stories` — 将功能拆分为待办事项 — 用户故事、工作故事或 WWA 格式，包含验收标准。
