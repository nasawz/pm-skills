# pm-product-discovery

产品经理的产品探索技能：创意构思、实验设计、假设验证、功能优先级排序和客户访谈总结。

## 技能（13个）

- **analyze-feature-requests** — 通过主题、战略对齐度、影响力、工作量和风险来分析和排定功能请求列表的优先级。
- **brainstorm-experiments-existing** — 为现有产品设计实验，以验证假设。
- **brainstorm-experiments-new** — 为新产品设计精益创业实验（原型验证）。
- **brainstorm-ideas-existing** — 采用多视角创意构思（产品经理、设计师、工程师）为现有产品构思产品创意。
- **brainstorm-ideas-new** — 为初期探索阶段的新产品构思功能创意。
- **identify-assumptions-existing** — 从价值、可用性、可行性和可行性四个维度，识别现有产品中功能创意的风险假设。
- **identify-assumptions-new** — 从8个风险类别（包括市场推广、战略和团队）识别新产品创意的风险假设。
- **interview-script** — 创建结构化的客户访谈脚本，包含JTBD探索问题、热身环节、核心探索和总结环节。
- **metrics-dashboard** — 定义并设计产品指标仪表板，包含关键指标、数据源、可视化类型和告警阈值。
- **opportunity-solution-tree** — 构建机会解决方案树（OST）来结构化产品探索——将期望成果映射为机会、解决方案和实验。
- **prioritize-assumptions** — 使用影响力×风险矩阵对假设进行优先级排序，并为每个假设建议实验。
- **prioritize-features** — 根据影响力、工作量、风险和战略对齐度对功能创意待办事项进行优先级排序。
- **summarize-interview** — 将客户访谈记录总结为结构化模板，包含JTBD、满意度信号和行动项。

## 命令（5个）

- `/pm-product-discovery:brainstorm` — 从产品经理、设计师和工程师的角度进行产品创意或实验头脑风暴——适用于现有产品或新产品。
- `/pm-product-discovery:discover` — 运行完整的产品探索周期——从创意构思到假设映射，再到实验设计。
- `/pm-product-discovery:interview` — 准备客户访谈脚本或将访谈记录总结为结构化洞察。
- `/pm-product-discovery:setup-metrics` — 设计产品指标仪表板，包含北极星指标、输入指标、健康指标和告警阈值。
- `/pm-product-discovery:triage-requests` — 分析、分类和优先级排序来自客户或利益相关者的一批功能请求。
